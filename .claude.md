# Claude Project Rules

## Core Instructions
- Use `system-architecture` as the main source for roadmap and feature priority.
- Continue feature development sequentially based on business flow.
- Finish one feature point, verify it, then commit it.

## Commit Policy
- Every feature point must be committed separately.
- Commit messages must be written in Chinese.
- Keep each commit focused on the current feature only.

## Documentation Policy
- Update `doc/development-log.md` after each feature point.
- Record what was implemented, how it was validated, and what should be done next.

## Testing Policy
- Validation is required for every feature point.
- Do not include newly written test source files in future commits unless the user asks for them.
- Use JDK 21 when running Maven verification for this repository.

## Scope Control
- Preserve existing project patterns.
- Avoid unrelated refactors.
- Ignore untracked support files unless they are part of the current feature.
